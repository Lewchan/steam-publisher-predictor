What This Builds

A transparent scoring-and-sales calculator, not a black box.

The project fetches Steam metadata, estimates game quality from ratings and discussion proxies, maps the reachable audience through a genre-pool table, and then produces a sales estimate with visible intermediate values. The target use case is analyst-grade personal decision support, not a public-facing SaaS product.

Current Flow
  • Input a Steam game name, URL, or app id
  • Fetch store metadata, review summary, and Steam tags
  • Estimate quality score and confidence
  • Map tags into audience pools with overlap adjustment
  • Calculate CL and predicted sales
Model Layers
  • Quality scoring from rating, proof, discussion, and persistence
  • User pool estimation from table-driven genre buckets
  • CL score from weighted linear model with hard cap
  • Sales output from conversion baseline times strength multiplier
Core Equation
sales = user_pool * exposure * intent * purchase * (1 + cl)^3 cl = min(3.0, max(0, b + Σ(w * x)))
