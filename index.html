项目定位

一个给发行研究使用的透明分析工具，不是黑盒结论机。

这个项目会抓取 Steam 公开信息，对游戏质量、关注度、用户池与发行强度进行拆解分析， 最终形成可解释的销量预估结果。它服务的是个人研究、发行判断与模型迭代，不追求表面上的“神准”， 而强调过程可复查、口径可调整。

当前流程
  • 输入游戏名、Steam 链接或 app id
  • 抓取商店元数据、评论摘要与标签
  • 综合估算质量与热度信号
  • 映射品类池并估算有效用户池
  • 输出销量预估与中间拆解结果
  • 支持保守/基准/乐观场景对比
建设方向
  • 继续补强质量评分与标杆校准
  • 继续扩展用户池映射与跨品类逻辑
  • 继续接入更多公开讨论与热度来源
  • 继续迭代预测口径与案例对照
页面说明

这个 GitHub Pages 页面作为前端调试壳，可以直接请求已部署的后端 API（`/api/analyze`）进行实时分析。 交互式预测器完整功能仍在 Streamlit 本地版本中持续迭代，包含完整的 Slider 表单、质量置信度警告、 SteamDB 数据展示与基准对比页面。

部署说明

当前 GitHub Pages 站点是静态项目页，用来承载项目介绍与后续迭代展示。交互式预测器仍在本地运行， 因为 GitHub Pages 本身不负责执行 Python 后端。后续如果要在线使用，再单独处理运行环境与服务部署。
